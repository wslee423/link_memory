# RELIABILITY.md

## 핵심 원칙
"조용한 실패 금지" — 모든 실패는 사용자에게 명확히 알린다.

## 단계별 실패 처리
| 단계 | 처리 |
|------|------|
| URL 저장 실패 | toast 에러 + 입력값 유지 |
| 메타데이터 수집 실패 | 제목=URL, 썸네일=기본 이미지 |
| 자막 수집 실패 | AI 요약 스킵, "자막 없음" 표시 |
| AI 요약 실패 | 스켈레톤 해제 + "요약 생성 실패 - 재시도" |
| 메모 저장 실패 | "저장 실패 - 재시도" |
| 인증 만료 | /login 리다이렉트 |

## 로딩 상태
| 작업 | UI |
|------|-----|
| 링크 저장 | 버튼 비활성화 + 스피너 |
| 메타데이터 수집 | 카드 shimmer 스켈레톤 |
| AI 요약 생성 | 요약 영역 shimmer → 완료 후 fade-in |
| 메모 저장 | "저장 중..." → "저장됨 ✓" |

## 외부 API
- OpenAI: 타임아웃 30초, 실패 시 ai_summary=null
- YouTube Data API v3: 일일 10,000건 쿼터 (초과 시 날짜 스킵)
- 자막 실패 시 링크 저장 자체는 완료 처리

## 성능 목표
| 항목 | 목표 |
|------|------|
| 링크 저장 응답 | < 500ms |
| 목록 로딩 | < 1초 |
| AI 요약 생성 | < 15초 (비동기) |
| 페이지 초기 로딩 | < 2초 |
